# Cross-Project Ideas

Idea capture space for concepts that aren't tied to a specific project.

Ideas here are project-agnostic -- general product concepts, experiments to try,
or cross-cutting concerns that span multiple projects.

## Where ideas go

- **Here** (`~/Development/ideas/`): Cross-project or unassigned ideas
- **Project-specific**: `<project>/workspace/ideas/` for ideas tied to a project

## File format

Use markdown files. Name them descriptively: `real-time-collaboration.md`,
`unified-auth-layer.md`, etc. No strict template -- capture the thought
while it's fresh.
